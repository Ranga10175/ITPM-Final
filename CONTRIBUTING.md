# Contributing to Hostel Management System

Thank you for your interest in contributing to the Hostel Management System!

## Getting Started

1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Ensure your code follows the existing style and patterns.
4. Submit a pull request with a clear description of your changes.

## Branching Strategy

- `main`: Production-ready code.
- `Develop`: Integration branch for new features.
- `feature/*`: Specific features or improvements.

## Development Workflow

1. Install dependencies in both `Backend` and `hostalezone` folders.
2. Use `npm run dev` for the frontend and `npm run dev` (via nodemon) for the backend.
